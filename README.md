# Assignment 1
## Project Overview
So this project is Banking Fraud Monitoring System. It stores data like customers, accounts, cards and transactions, and detects suspicious activity using triggers, functions and procedures.


 ## Project Structure
 
```
01_ddl_tables.sql - tables creation
02_functions.sql - my functions
03_procedures.sql - my procedures
04_triggers.sql - triggers
05_views.sql - views
06_materialized_views.sql - materialized view
07_sample_data.sql - generated by claude data
08_demo_queries.sql - my demo queries
```

## Setup Instructions

1. Create a new database:
```sql
CREATE DATABASE fraud_monitoring;
```
2. Connect to database and run all my scripts in order by number in their names

 
## Fraud Detection Logic
 
Transaction is flagged as suspicious when:
- Its amount is greater than 30,000
- Its country is in the high-risk country list (RU, KP, IR, SY, BY)

## Materialized Views Refresh Strategy
 
Materialized views do not update automatically when data changes. To refresh them manually we need to:
 
```sql
REFRESH MATERIALIZED VIEW mv_country_risk_summary;
```

For automatic refresh `pg_cron` extension can be used. This would refresh the view every hour.
